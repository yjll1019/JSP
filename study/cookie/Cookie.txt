1. 인터넷에서 사용 가능한 웹 서비스는 FTP , HTTP 등이 있다.
  -FTP(File Transfer Protocool) : 원격 서버 사이에 파일을 교환하기 위한 TCP/IP 프로토콜
  -HTTP(HyperText Tansfer Protocool) : HTML 문서의 송수신을 위해 사용하는 프로토콜 
2. HTTP  비 연결 지향형 통신 프로토콜 
 - 클라이언트가 서버에 정보를 요청하면 해당 클라이언트에 전송한 후 연결을 끊는다.
 - 따라서 이전 정보를 기억하지 못한다.
 - 서버의 자원 낭비 최소화한다.
3. HTTP의 이런 단점을 극복하기 위해 사용되는 쿠키(cookie)
 - 클라이언트 컴퓨터의 하드디스크에 사용자의 정보를 저장한다.
 - 서버의 부하를 줄일 수 있음.
 - 쿠키에 대한 정보가 필요한 웹 페이지가 서버에 요청할 때 쿠키에 대한 정보를 웹 서버에 보낸다.
 - 크기가 제한되기 때문에 오래된 정보는 자동으로 삭제된다.
 - 이미지는 용량이 크기 때문에 경로만 저장된다.
4. 생성한 모든 쿠키 출력 시 JSESSIONID 
 - 세션 ID는 쿠키를 사용해서 유지되는데 이때 이 쿠키의 이름이 JSESSIONID이다.
5. 쿠키의 단점
 - 보안 유지가 되지 않는다.
 - 데이터의 제한
6. 쿠키의 단점을 극복하기 위해 사용되는 세션(Session)
 - JSP(서버)에서만 접근 가능하기 때문에 보안 유지에 강력하다.
 - 데이터의 한계가 없다.
 - 브라우저 당 한 개씩 존재한다.
